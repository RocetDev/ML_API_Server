Конфигурация доступа к моделям машинного обучения

Является важной частью структуры сервера без которого ничего не будет рабоать. В этом файле подключаются все работоспособные модели машинного обучения. Вот тестовый пример:

import Models.Model_one.run as model_one

models_names = [
'Model_one'
]

model_predict = {
'Model_one': model_one,
}

Сначало импортируется модель, а имеено обязательный фаил run.py в котором есть функция predict(). На вход ей подается словарь полученный из аргументов URL, а возвращает результат работы нейросети. В Список models_names добавляется имя модели. Это сделанно для проверки существования модели и обновления списков доступных моделей. В словрь model_predict добавляется моедль с импортированных пакетом, где лежит файл run.py

Конфигурация пакета

В папке Models находятся все используемые модели. Каждая модель должна хранится в отдельной папке. В самой же папке должен, в обязательном порядке, находится файл run.py. В файле должна быть объявлена функция predict() принимающая на вход словарь. Функция описывает взаимоделйствие с моделью, хранящейся в этой же директории. Вот наглядный пример иерархии файлов:

  • Model_one
    1. __init__.py
    2. run.py
    3. Дополнительные файлы и код
  • Model_two
    1. __init__.py
    2. run.py
    3. Дополнительные файлы и код
  • ...
